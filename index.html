RPH SLASH PROJECT — Version 1.7

About

RPH SLASH PROJECT is a command-driven toolkit implemented as an interactive CLI designed for structured reconnaissance, data enrichment, and a variety of specialized utilities. The codebase exposes modular routines for identity parsing, web information harvesting, administrative path enumeration, image processing, signature operations, IP intelligence, XSS checking, automated vulnerability scanning, onion-network querying, and a number of auxiliary utilities. Each command invokes a self-contained Python routine that collects, formats, and prints findings to the console for analyst consumption, Other projects are not public for legal reasons, check https://github.com/faelll14 to see all published projects.

Available Commands & Descriptions

rph -out
Exits the CLI session and terminates the program.

rph -nik
Runs a National Identification Number (NIK) parsing routine. The command reads a 16-digit NIK, validates its format, invokes an external parser (`nik-parse`) when available, and prints structured attributes such as birthdate, sex, region, derived age, zodiac, and additional metadata. The routine includes basic input validation and graceful error handling.

rph -apf
Executes an administrative path finder. The module accepts a target URL, normalizes the domain, and enumerates a comprehensive list of common administrative and management paths using concurrent worker threads. It issues HTTP requests with timeouts, reports HTTP status codes for discovered endpoints, and presents progress feedback while scanning.

rph -cweb
Performs an extensive website information audit. The implementation requests the target URL, parses HTML with BeautifulSoup, and extracts structured data including page title, meta description, HTTP status, domain/IP resolution, DNS A records, ISP/hosting metadata (via ipinfo when available), internal/external link counts, Open Graph and Twitter tags, JavaScript/CSS/image inventories, headers, cookies, robots/sitemap availability, SSL certificate details (when HTTPS), canonical/robots tags, content-size metrics, CMS detection heuristics, form counts, and a set of derived performance and security indicators. Results are printed as labeled fields for analyst review.

rph -cc
Invokes the CC checker module. This command is present in the CLI routing table and calls a dedicated function intended to validate or test card-related inputs according to the codebase's implementation. The function is bound to the CLI and prints results to standard output.

rph -cipsc
Runs the ciphertext scope utility. The CLI mapping forwards to a ciphertext-scoping routine which is intended to examine or process ciphertext-related inputs or artifacts and present scoped results; the command is exposed in the router and implemented as a separate function in the codebase.

rph -imac
Launches the image-to-ASCII converter. This routine loads a PNG/JPG via Pillow, flattens alpha into a white background, rescales to terminal width, converts pixel luminance to a character palette, emits ANSI 24-bit color per character to render colored ASCII art on-screen, and writes a Python `.py` file that reproduces the ASCII output. The command performs file existence checks and provides user prompts and output confirmation.

rph -sign
Opens the signature utility menu. The feature set accessible from this menu includes Ed25519 signing and signature verification options. The menu provides interactive choices for signing a file or verifying an existing signature and returns to the main CLI on completion.

rph -ipc
Runs the IP information checker. This routine resolves a host to an IP address, prints lookup results, and may query external IP intelligence services where implemented to collect ISP, ASN, geolocation, and related metadata. Results are presented in labeled fields.

rph -xss
Executes a reflected XSS checking routine. The command triggers a scanner intended to probe pages for reflected cross-site scripting vectors across discovered URLs. The function is mapped in the router and reports findings per page scanned.

rph -vuln
Invokes the automated vulnerability checker. The CLI calls a module designed to perform automated checks against target surfaces and emit vulnerability findings. The routine aggregates and prints discovered issues according to its internal checks.

rph -onion
Starts the onion-network checker and searcher. The routed function queries and inspects .onion resources via the module mapped to this command and prints search or enumeration results returned by that routine.

rph -virus
Calls the generator routine present in the codebase. The router maps this command to a `gen()` function which, according to the implementation, writes platform-targeted files when invoked. The codebase contains a generation menu with selectable payload templates and file-writing behavior; exercise extreme caution with this routine and review the implementation before use.

Notes
All listed commands are exactly those exposed by the script's `command_router` and help text. Each command is implemented as an internal Python function; descriptions above reflect the observed behavior and outputs present in the code. The project prints progress indicators and formatted result fields to the console rather than returning structured networked outputs.
